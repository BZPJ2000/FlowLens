# PoltAIshow 项目需求总结

## 项目定位
代码数据流可视化平台 — 将代码项目的文件间数据流关系以可视化图形方式呈现，帮助开发者理解代码结构和数据传输过程。

---

## 核心可视化需求

### 1. 布局方式：弹性自适应（Flex-wrap）
- 每个文件是一个卡片（框），卡片大小根据内容自适应
- 卡片不是固定的长条形，而是可以任意填充比例
- 卡片按拓扑排序（数据依赖顺序）从左到右排列，超出画布宽度自动换行
- 同一行的文件是并行关系（互不依赖），不同行是串行关系（上面被下面依赖）
- 像流程图一样，有的串行（上下排列），有的并行（左右排列）

### 2. 文件卡片结构
- 每个文件卡片 = 一个封装好的模块
- 左侧：输入端口（接收的变量）
- 右侧：输出端口（导出的变量）
- 中间：内部函数/类的网格布局
- 头部：文件名、语言、架构角色标签
- 底部：统计信息

### 3. 连线规则（最重要）
- **跨文件连线**：只能连接文件节点的端口 ↔ 端口，不能直接连到内部函数
- **文件内部连线**：端口 → 函数参数、函数输出 → 端口
- **每个参数独立连线**：一个函数接收多少个变量就有多少条线连到它身上
- **连线不能重叠**：多条平行边必须错开显示
- **四方向连接**：上下左右都可以是连线接点，根据节点相对位置自动选择最短路径
- **连线标签**：每条线上要有标签框，清晰显示传递的变量名和类型
- **方向明确**：从哪个文件的哪个变量传到哪个文件的哪个变量，顺序方向要清楚

### 4. 变量名和连线是最主要的
- 变量名要显眼、清晰
- 连线要粗、有颜色区分（按数据类型着色）
- 每个函数要标注清楚：导入哪些值、输出哪些值、和下一个函数怎么连接

### 5. 函数/类显示规则
- 过滤掉 `__init__`、`constructor` 等构造/生命周期函数（前端过滤）
- 类节点要包含住内部方法，不能让方法重叠
- 类容器大小要根据方法数量和参数数量动态计算
- 每个方法和函数一样，有独立的参数 Handle 和输出 Handle

### 6. 类型颜色系统
- string = 蓝色
- number = 绿色
- boolean = 琥珀色
- object = 紫色
- function = 粉色
- Promise = 天蓝色
- array = 橙色
- void/unknown = 灰色

## 动画播放功能

### 需求
- 一键启动按钮
- 从第一个启动脚本开始，按 1→2→3→4... 顺序走到最后一个处理脚本
- 高亮当前脚本的边框
- 连线上有流动动画表示数据传输方向
- 其他未激活的节点和连线变暗
- 可以随时停止

---

## 历史记录功能

### 需求
- 显示所有已分析的项目列表
- 每个项目显示：名称、来源类型、文件数、时间、分析状态
- **删除功能**：可以删除单条历史记录，清理测试数据
- 点击已完成的项目可以直接查看可视化结果

---

## 交互功能

### 已有
- 节点选中高亮（点击）
- 路径追踪（双击节点，高亮整条数据流路径，5秒后恢复）
- 搜索过滤（按文件名/函数名搜索，匹配项高亮，不匹配变暗）
- PNG 导出
- 详情面板（点击节点查看文件详情）
- AI 聊天面板
- 图例面板

---

## 技术栈

### 前端
- React 18 + TypeScript + Vite
- ReactFlow v12（节点图可视化库）
- Zustand（状态管理）
- Tailwind CSS（样式）
- Lucide React（图标）

### 后端
- FastAPI (Python)
- SQLAlchemy + AsyncSession
- Tree-sitter（代码解析）
- AI 分析（文件级别的输入/输出/架构角色识别）

### 数据流
1. 用户导入项目（GitHub URL 或上传 ZIP）
2. 后端 Tree-sitter 解析代码结构（函数、类、import/export）
3. AI 分析每个文件的输入输出和架构角色
4. GraphBuilder 构建数据流图（节点 + 边）
5. 前端 layout.ts 计算布局位置
6. ReactFlow 渲染可视化

---

## 视觉风格
- 深色主题（ComfyUI 风格）
- 干净简洁
- 端口用彩色圆点表示数据类型
- 连线有发光效果 + 流动动画
- 节点有微妙的阴影和渐变

---

## 当前存在的问题（未完全解决）

1. 连线仍有重叠现象，需要更智能的路由算法
2. 类容器内方法可能重叠，动态尺寸计算需要更精确
3. 跨文件连线不应该直接连到内部函数（已修复逻辑，需验证效果）
4. 整体排版需要保证连线清晰可见，不被节点遮挡
5. 动画播放的拓扑顺序应该用 layout.ts 计算的真实拓扑序，而不是简单的节点列表顺序
